# RoVoCar CSV Maker PRD

- 문서 상태: 개발 전 작업 지시서
- 작성일: 2026-07-19
- 프로젝트 폴더: `projects/RoVoCar_CSV_Maker/`
- 제품명(가칭): **RoVoCar CSV Maker**
- 대상: RoVoCar에 단어장을 올릴 가족 사용자

## 1. 문제와 목표

RoVoCar는 `English,Korean` 형식의 CSV를 가져와 단어장을 만든다. 현재는 단어장 사진을 개인 ChatGPT/Codex 대화에 올려 CSV 생성을 요청해야 해서, 가족 구성원이 독립적으로 사용하기에는 단계가 많고 결과 형식도 매번 확인해야 한다.

이 서비스는 종이 영어 단어장 사진 한 장을 받아 Gemini API로 영어 스펠링과 한글 뜻을 추출하고, 사용자가 검토·수정한 결과를 RoVoCar 호환 CSV로 다운로드하게 한다.

### 목표

- 휴대폰에서 사진 선택부터 CSV 다운로드까지 한 흐름으로 끝낸다.
- 다운로드한 파일을 RoVoCar에서 그대로 불러올 수 있게 한다.
- Gemini API 키를 브라우저·Git 저장소·로그에 노출하지 않는다.
- 사진과 추출 결과를 기본적으로 서버 DB/스토리지에 저장하지 않는다.
- 가족 사용자가 OCR 오류를 발견하고 직접 고칠 수 있게 한다.

### 성공 기준(MVP)

1. 사용자가 사진 한 장을 업로드하고 3분 안에 CSV를 다운로드할 수 있다.
2. 생성 CSV의 첫 행은 정확히 `English,Korean`이고, 쉼표·따옴표를 올바르게 이스케이프한다.
3. RoVoCar의 현재 CSV 파서에서 파일을 불러왔을 때 단어·뜻이 뒤바뀌거나 행이 깨지지 않는다.
4. Gemini 응답 실패, 사진 문제, 네트워크 오류에 사용자가 이해할 수 있는 다음 행동을 안내한다.
5. API 키·원본 사진·추출 전문이 브라우저 콘솔, 분석 로그, Git 저장소, 일반 DB 테이블에 남지 않는다.

### 비목표(MVP에서 하지 않을 일)

- RoVoCar 단어장에 직접 업로드하거나 RoVoCar DB를 수정하는 기능
- 여러 장 PDF/사진을 한 번에 처리하는 기능
- OCR 결과를 자동으로 정답이라 판단하는 기능
- 학습 기록, 단어장 관리, 퀴즈 기능
- 원본 사진·CSV·OCR 결과의 서버 보관함
- 공개 불특정 다수를 위한 무제한 API 제공

## 2. 사용자와 핵심 시나리오

### 주요 사용자

- 아이: 숙제나 학원 단어장을 스스로 RoVoCar에 넣고 싶은 사용자
- 보호자: 사진을 찍어 가족의 단어장 CSV를 준비하는 사용자
- 운영자: Gemini 사용량·오류만 최소한으로 확인하고 싶은 관리자

### 대표 사용자 흐름

1. 사용자가 모바일 브라우저에서 서비스를 연다.
2. 카카오 로그인을 한다. 기존 RoVoCar 가족 계정을 재사용하는 것을 기본으로 한다.
3. `사진 찍기 또는 선택`을 눌러 단어와 한글 뜻이 함께 보이는 사진 한 장을 고른다.
4. 서비스가 사진 품질 안내와 함께 업로드 전 미리보기를 보여 준다.
5. 사용자가 `단어 추출하기`를 누른다.
6. 서버 함수가 인증·사용량 한도를 확인한 뒤 Gemini API에 사진과 구조화된 추출 지시를 보낸다.
7. 화면에 영어 / 한글 뜻 표 형태의 초안과 경고 행을 보여 준다.
8. 사용자가 오인식, 빈 뜻, 줄이 합쳐진 행을 고치거나 삭제·추가한다.
9. 사용자가 `CSV 다운로드`를 누른다.
10. 브라우저가 UTF-8 CSV를 내려받는다. 화면은 이어서 RoVoCar에서 `CSV 불러오기`를 누르는 3단계 안내를 제공한다.

## 3. 제품 범위와 요구사항

### 3.1 화면 구성

| 화면/상태 | 한 가지 목적 | 필수 요소 |
| --- | --- | --- |
| 시작 | 서비스를 이해하고 로그인 시작 | 서비스 설명, 개인정보 요약, 카카오 로그인 |
| 사진 선택 | 올바른 사진 선택 | 카메라/파일 선택, 지원 형식·용량 안내, 촬영 팁 |
| 사진 미리보기 | 전송 전 이미지 확인 | 회전, 다시 선택, 추출 시작 버튼 |
| 추출 중 | 대기 상태를 분명히 알림 | 진행 문구, 중복 전송 방지, 취소/되돌아가기 안내 |
| 결과 검토 | OCR 결과를 고쳐 CSV 준비 | 영어·뜻 편집 표, 행 추가/삭제, 행별 경고, 단어 수 |
| 다운로드 완료 | RoVoCar로 이어지게 함 | 다운로드 확인, RoVoCar 업로드 3단계, 새 사진으로 시작 |
| 오류 | 사용자가 복구할 다음 행동 제시 | 원인에 맞는 안내, 재시도/사진 교체/나중에 시도 버튼 |

모바일 우선으로 설계한다. 첫 화면의 목적은 사진을 올리는 것 하나이며, 결과 화면의 목적은 CSV를 내려받기 전 정확히 고치는 것 하나다. 일반적인 AI SaaS 대시보드·가짜 통계·과한 카드 UI는 피한다.

### 3.2 사진 입력

- MVP는 사진 한 장만 받는다.
- 허용 형식의 초기 권장안은 JPEG, PNG, WebP이며, 정확한 Gemini 지원 형식과 서버 함수 제한은 구현 직전 공식 문서로 확인한다.
- iPhone의 HEIC 사진은 환경별 호환성이 다를 수 있으므로, MVP에서는 명확한 오류와 JPEG로 다시 선택하는 안내를 제공하거나 클라이언트 변환을 별도 검증 후 지원한다. 지원 여부를 추측으로 처리하지 않는다.
- 최대 파일 크기는 배포 플랫폼·Gemini 요청 제한을 확인한 뒤 정한다. 초기 UX에는 `사진을 선명하게, 한 장씩`이라는 안내를 둔다.
- 업로드 전 미리보기에서 다음을 안내한다: 종이가 사각형에 가깝게 보이는지, 글자가 흐리지 않은지, 그림자/손가락이 단어를 가리지 않는지, 영어와 뜻이 모두 찍혔는지.
- 사진은 요청 처리 중 서버 함수 메모리에만 머무르게 하고, Storage·DB·분석 도구에 원본을 보관하지 않는다.

### 3.3 Gemini 추출 계약

Gemini에는 자연어 CSV가 아니라 아래 구조의 JSON만 반환하도록 요청한다. 구현 시 선택 모델명, 요청 URL, JSON 응답 옵션, 이미지 전달 방식, 무료 할당량은 **Gemini 공식 문서로 다시 확인**한다.

```json
{
  "items": [
    {
      "english": "take care of",
      "korean": "돌보다",
      "confidence": "high",
      "note": null
    }
  ],
  "warnings": [
    "사진 하단의 한 행은 글자가 잘려 확인이 필요합니다."
  ]
}
```

프롬프트 요구사항:

- 사진에 실제로 읽히는 영어 단어/구와 대응 한글 뜻만 추출한다.
- 번호, 페이지 제목, 예문, 발음기호, 장식 문구는 단어 행으로 만들지 않는다.
- 영어와 뜻의 순서를 원본과 동일하게 유지한다.
- 확신하지 못한 값은 추정해 채우지 말고 `confidence: "low"` 또는 `note`로 표시한다.
- 영문 철자와 한글 뜻이 뒤바뀌지 않게 한다.
- Markdown, 코드 펜스, 설명문 없이 정의된 JSON 스키마만 반환한다.

Gemini 응답은 신뢰할 수 없는 외부 입력으로 취급한다. 서버와 클라이언트 양쪽에서 JSON 스키마·문자열 길이·배열 길이를 검증하고, 화면에 렌더링할 때 HTML로 삽입하지 않는다.

### 3.4 검토·편집 규칙

- 각 행은 `영어 스펠링`과 `한글 뜻`을 독립적으로 편집할 수 있다.
- 행 추가, 행 삭제, 빈 행 제거를 제공한다.
- 영어 또는 뜻이 비어 있거나 Gemini가 낮은 확신으로 표시한 행에는 눈에 띄는 확인 표시를 둔다.
- 다운로드 전 빈 영어/뜻 행은 자동 제외하되, 제외 개수를 사용자에게 알린다.
- 뜻이 여러 개인 경우 한 셀에 `수반하다, 포함하다`처럼 유지한다.
- 단어 중복은 OCR 결과 화면에서 경고만 하며, 자동 삭제하지 않는다. RoVoCar는 새 단어장 등록 시 중복을 허용한다.
- 사용자가 수정한 검토 결과는 페이지를 새로고침하면 사라지는 것이 기본이다. 복구 기능은 후속 범위다.

### 3.5 CSV 생성 계약 (RoVoCar 호환)

다운로드 파일은 아래 규칙을 반드시 만족한다.

```csv
English,Korean
apple,사과
take care of,돌보다
"right, correct",옳은
involve,"수반하다, 포함하다"
```

- 첫 행은 정확히 `English,Korean`이다.
- 첫 번째 열은 영어, 두 번째 열은 한글 뜻이다.
- UTF-8 인코딩으로 생성한다. Excel 호환을 위해 UTF-8 BOM을 넣을지는 실제 iPhone·RoVoCar·Excel 검증 결과를 기준으로 결정하고 테스트한다.
- 셀 안에 쉼표, 큰따옴표, 줄바꿈이 있으면 RFC 4180 방식으로 큰따옴표로 감싸고 내부 큰따옴표는 두 번 쓴다.
- 파일명 예: `rovocar-words-2026-07-19.csv`. 사용자가 단어장 이름을 입력하면 안전한 파일명으로 반영할 수 있다.
- 실제 RoVoCar의 `core.mjs` 파서와 동일한 케이스로 생성 결과를 검증한다. RoVoCar의 현재 파서는 헤더를 제외하고 첫 열을 영어, 나머지 열을 쉼표로 합친 뜻으로 읽는다.

### 3.6 로그인·사용량 제어

무료 Gemini 할당량과 가족 전용 목적을 보호하기 위해, OCR 실행은 로그인한 사용자만 할 수 있게 한다.

- RoVoCar에서 사용 중인 Supabase Auth + 카카오 로그인을 재사용한다.
- 새 서비스의 운영 URL과 `http://localhost` 개발 URL을 Supabase Auth 리디렉션 허용 목록에 추가한다.
- 서비스는 RoVoCar의 단어장·학습 데이터 테이블을 읽거나 수정하지 않는다.
- 사용량 기록은 `user_id`, 일자, 요청 횟수, 최근 요청 시각 정도로 최소화한다. 이미지 원본, OCR 원문, 이메일, 인증 토큰을 사용량 테이블에 기록하지 않는다.
- 초기 한도는 구현 전에 Gemini 무료 할당량을 확인해 결정한다. 예: 사용자별 일일 N회, 분당 M회. 한도 값은 코드 상수가 아니라 환경 설정 또는 서버 설정으로 관리한다.
- 사용량 제한은 반드시 서버 함수와 DB의 원자적 처리로 적용한다. 브라우저의 버튼 비활성화만으로 제한하지 않는다.

### 3.7 오류와 상태 문구

| 상황 | 사용자 안내의 핵심 | 사용자 행동 |
| --- | --- | --- |
| 사진 형식/용량 불가 | 지원 사진으로 다시 선택하도록 설명 | 사진 다시 선택 |
| 사진이 흐리거나 구조가 불명확 | 결과는 초안이며 직접 검토가 필요함을 안내 | 사진 교체 또는 결과 편집 |
| 네트워크 단절 | 사진을 전송하지 못했음을 명확히 표시 | 연결 후 재시도 |
| Gemini 일시 실패/시간 초과 | 저장된 결과가 없음을 알리고 잠시 뒤 재시도 안내 | 재시도 |
| 일일 한도 도달 | 가족용 사용량 한도임을 간단히 설명 | 다음 날 재시도 |
| 인증 만료 | 다시 로그인해야 함을 안내 | 로그인 |
| 응답 형식 검증 실패 | 추출 결과를 안전하게 표시하지 않았음을 알림 | 재시도 또는 사진 교체 |

오류 메시지에 Gemini API 키, HTTP 전문, 토큰, 서버 내부 오류 원문을 표시하지 않는다.

## 4. 권장 기술 아키텍처

### 결정: 정적 프런트엔드 + Supabase Edge Function

RoVoCar와 같은 Supabase 프로젝트의 인증을 재사용하되, OCR 호출은 별도 Supabase Edge Function으로 분리한다. 정적 웹앱은 GitHub Pages 등으로 배포할 수 있고, Gemini API 키는 Supabase Edge Function의 비밀 환경 변수에만 둔다.

```text
모바일 브라우저
  ├─ Supabase Auth (카카오 로그인)
  ├─ 사진 미리보기 · 결과 편집 · CSV 다운로드
  └─ 로그인 JWT + 사진  ────────────────────────┐
                                                     ▼
Supabase Edge Function: extract-vocabulary
  ├─ JWT 검증
  ├─ DB 원자적 사용량 제한
  ├─ GEMINI_API_KEY (Supabase Secret, 서버 전용)
  ├─ Gemini API에 이미지 + 구조화 프롬프트 전송
  └─ 검증한 JSON만 브라우저에 반환
                                                     │
                                                     ▼
                                              Gemini API
```

### 필수 보안 경계

- `GEMINI_API_KEY`는 `.env`에만 두고 Git에 추가하지 않는다. 프런트엔드 JavaScript, 정적 HTML, GitHub Actions 로그, 오류 메시지에 절대 넣지 않는다.
- Supabase의 publishable/anon key는 기존과 같이 클라이언트에 둘 수 있지만, `service_role`/secret key는 두지 않는다.
- Edge Function은 CORS 허용 출처를 운영 도메인과 localhost 개발 주소로 제한한다.
- Edge Function은 인증되지 않은 요청을 거부하고, 요청 본문 크기·MIME 타입·시간 제한을 검사한다.
- 브라우저가 보낸 파일명·MIME 타입·Gemini 응답을 신뢰하지 않는다.
- 서버 로그에는 요청 ID, 성공/실패 종류, 지연 시간, 사용량 카운트 같은 최소 운영 정보만 남기며 사진·OCR 텍스트·인증 헤더는 마스킹 또는 기록 제외한다.
- 세션 토큰은 URL 쿼리, localStorage 백업 파일, 오류 추적 도구의 사용자 식별값으로 복사하지 않는다.

### 배포 구성(초기안)

- 프런트엔드: 별도 GitHub 저장소와 GitHub Pages 또는 동등한 정적 호스팅
- 서버: 현재 RoVoCar Supabase 프로젝트의 Edge Function
- 인증/사용량: 현재 Supabase Auth 및 새 전용 테이블/함수/RLS
- 비밀값: Supabase Secrets에만 설정

새 서비스가 RoVoCar와 인증 기반을 공유하더라도, 코드 저장소·배포 주소·서비스 워커 캐시는 서로 분리한다.

## 5. 데이터 설계 초안

### 브라우저 메모리 상태

```text
selectedImage       선택한 사진(File/Object URL) — 페이지 종료 시 폐기
extractionResult    Gemini가 반환하고 검증된 행 배열 — 페이지 종료 시 폐기
editedRows          사용자가 편집한 행 배열 — 페이지 종료 시 폐기
```

### 서버 DB: 최소 사용량 테이블 예시

`ocr_usage_daily`

| 필드 | 용도 | 보관 원칙 |
| --- | --- | --- |
| `user_id` | 사용자별 한도 판정 | Supabase Auth UUID, RLS로 본인/함수만 접근 |
| `usage_date` | 일일 한도 판정 | 날짜 |
| `request_count` | 요청 횟수 | 정수 |
| `updated_at` | 운영 점검 | 시각 |

- 기본키: `(user_id, usage_date)`
- 클라이언트는 직접 수정할 수 없다.
- Edge Function에서만 호출할 수 있는 DB 함수가 한도 확인과 증가를 한 트랜잭션으로 수행한다.
- OCR 이미지, CSV 내용, 영어·한글 단어 전문은 이 테이블과 별도 테이블에 저장하지 않는다.

## 6. 접근성·개인정보 UX

- 파일 선택 버튼과 결과 편집 표에 명확한 레이블을 제공한다.
- 업로드/추출/다운로드 상태는 `aria-live`로 알린다.
- 키보드만으로 사진 선택 뒤의 결과 편집·행 삭제·CSV 다운로드를 완료할 수 있어야 한다.
- 오류가 발생하면 오류 제목으로 포커스를 이동하고, 어느 입력을 고쳐야 하는지 연결한다.
- 색상만으로 낮은 신뢰도·오류 행을 구분하지 않는다.
- 시작 화면에 `사진은 OCR 요청 처리 후 서버에 저장하지 않으며, 추출 결과는 이 브라우저에서 CSV 다운로드용으로만 사용됩니다.`라는 짧은 안내를 둔다.
- 단, Gemini API 제공자에게 사진이 전송된다는 사실과 해당 제공자의 데이터 처리 정책을 확인할 수 있는 링크를 개인정보 안내에 명시한다. 서비스 개인정보 처리방침은 실제 운영 전에 작성·게시한다.

## 8. 테스트 계획

### 자동 테스트

- CSV 이스케이프: 쉼표, 큰따옴표, 줄바꿈, 한글 복수 뜻
- 빈 영어/뜻 행과 공백 행 제거
- Gemini JSON 스키마: 정상·누락 필드·너무 긴 값·예상 밖 타입
- API 호출 전 인증/파일 형식/크기/요청 한도 거부
- 사용량 증가 동시 요청 경합: 한도를 넘겨 통과하지 않는지
- Edge Function 응답에서 비밀값·원본 오류 전문이 드러나지 않는지

### 수동 테스트

- 선명한 사진, 기울어진 사진, 그림자 있는 사진, 여러 열처럼 보이는 사진, 한 행이 잘린 사진
- `take care of`, `right, correct`, `수반하다, 포함하다` 같은 CSV 특수값
- iPhone 사진 선택/촬영, Safari 다운로드, Files에서 RoVoCar로 CSV 선택
- 로그인 만료, 오프라인, Gemini 일시 장애, 일일 한도 도달
- 키보드와 화면 낭독기 기본 흐름

## 9. 개발 세션에 전달할 시작 요청

```text
이 프로젝트는 RoVoCar CSV Maker다. 먼저 PRD.md 전체를 읽고, 구현 전 단계 0의 확인 항목과 현재 저장소 상태를 요약해줘.

이 서비스는 종이 영어 단어장 사진 한 장을 Gemini API로 추출하고, 사용자가 검토·수정한 English,Korean CSV를 다운로드하는 가족용 웹서비스다. RoVoCar DB에 직접 쓰지 않는다.

절대 조건:
- Gemini API 키, Supabase service role key, 인증 토큰, 원본 사진, OCR 전문을 Git·프런트엔드·로그·오류 메시지에 남기지 않는다.

- 사진과 OCR 결과는 기본적으로 서버에 영구 저장하지 않는다.
- 생성 CSV는 RoVoCar의 English,Korean 형식과 호환되어야 한다.
- 먼저 Gemini와 Supabase의 현재 공식 제약을 확인해 DECISIONS.md에 키가 아닌 선택 근거만 기록한다.

처음 구현은 Gemini 없이 더미 데이터로 사진 선택 → 결과 편집 → CSV 다운로드와 테스트를 완성해줘. 이후 각 단계는 테스트 결과와 git diff를 요약해줘.
```
